xfile. Загрузка файла

 

Поле для присоединения и отправки файла. Допускается множественное присоединение файлов.

 

Корректность введенного значения (совпадение с типом или расширением) проверяется на сервере. В случае ошибки форма возвращается пользователю для исправления.

 

         <control_group>xfile</control_group>

Пример.
 

<control>
        <control_group multy="true">xfile</control_group>
        <control_name>tst_xfile</control_name>
        <control_id>xfile_id</control_id>

         <filter_mime></filter_mime>

         <data_storage>store</data_storage>

         <allow_multy/>
        <file_ext>xfile,extensions,comma,separated</file_ext>
        <control_classes>foo</control_classes>
</control>

Элементы <control_id> и <control_classes> являются необязательными.

 multy

Атрибут показывает, возможна ли множественная загрузка файлов. По умолчанию - false. Возможные значения : true, false, <число файлов для загрузки>. В случае, когда переданы false или 1 - атрибут multy для файла не ставится.

<filter_mime>

Фильтр, который применяется для окна открытия файла для загрузки. Пока не поддерживается.

<data_storage>

Имя хранилища данных (см. Инициализация модуля создания форм).

<allow_multy>

Допускается ли мультипликация (размножение при помощи кнопки +) поля ввода. Пока не поддерживается.

Контроль файла на соответствие указанному типу осуществляется на стороне сервера. В случае несовпадения расширения файла с указанным в <file_ext> форма возвращается пользователю для исправления.

Поддерживаемые расширения а также максимально допустимый размер файла (соответствующий максимальному размеру POST-запроса) выводятся под соответствующей полю с файлом поясняющей надписью.

Сохранение загруженного файла происходит в хранилище данных после прохождения форматно-логического контроля.

Хранилище данных определяется именем из элемента <data_storage>, а его местоположение, в свою очередь, полем инициализации $.binary_path[] (См. Инициализация модуля создания форм).

Каталог хранилища данных создается автоматически и имеет следующую структуру.

$.data_storage__path/
|__ <control_name>/ 
  |___index.cfg 
  |___год/ 
    |______месяц/ 

              |_______день/

                |_______<UUID>.xcfile.<original_file_extension>

Поля ввода типа file имеют встроенный контроль максимального размера. В случае, если суммарный размер всех загруженных файлов превышаем максимальный размер POST, система выдает ошибку и соответствующее сообщение.
 
Важно!
Размер поля в базе данных (по умолчанию VARCHAR(255)) следует переопределить вручную, исходя из того, что на один файл необходимо 350 символов.
