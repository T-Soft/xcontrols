Пользовательские элементы

 

Класс-обработчик может быть дополнен модулями новых элементов управления.

 

Для использования в качестве обработчика модуль должен иметь то же имя, что и группа, к которой относится элемент управления.

При вызове модуля обработки, ему передается хеш со значениями параметров соответствующего элемента управления. Все значения полей хеша - строки.
 

Имя поля

Обязательное поле

Описание

$.group[]

да

Группа элементов. Совпадает с именем модуля обработчика.

$.type[]

нет

Тип элемента в группе.

$.name[]

да

Имя элемента для обращения. Совпадает с соответствующим полем базы данных.

$.id[]

нет

Идентификатор элемента для привязки скриптов.

$.label[]

да (кроме элементов hr,br)

Текстовое описание элемента для отображения.

$.value[]

нет

Начальное значение элемента.

$.properties[]

нет

Пользовательские свойства элемента для HTML верстки.

$.classes[]

нет

Пользовательские классы элемента для HTML верстки.

$.sql_query[]

нет

SQL запрос для выборки значений поля из базы данных.

$.required[]

нет

Выставляется в true в случае обязательности поля. В ином случае отсутствует.

$.bounds[]

нет

Границы диапазона вводимого значения. Представляет из себя откомпилированный атрибут data-.

$.file_type[]

только для элемента <file>

Тип файла для проверки передаваемых данных через элемент <file>.

$.file_ext[]

только для элемента <file>

Расширение файлов для проверки передаваемых данных через элемент <file>.

$.form_id[]

да

Идентификатор формы, на которой располагается элемент. Совпадает с директорией размещения описательного XML.

(!)В модуле-обработчике с именем $name$ обязательно  должны быть реализованы методы со следующими именами и назначениями.

 

@create_$name$[hData]

       "конструктор"  элемента управления. Описывает создание самого компонента.

 

@fieldnames_table_$name$[name]

        Получение имен полей элемента управления для внесения информации в базу данных. Тип возвращаемого значения - таблица.

 

@db_preprocess_$name$[value]

        Контроль содержимого поля и преобразование для внесения его в базу данных. Тип возвращаемого значения - хеш.

 

@edit_preprocess_$name$[init;hParsed_xml]

        Преобразование хранящегося в базе данных содержимого для отображения в качестве значения элемента управления. Тип возвращаемого значения - строка.

@ErrorList_$name$[invoke]

        Список ошибок, вызываемых данным модулем. Используется системой контроля внутренних ошибок и для показа пользователю. Тип возвращаемого значения - хеш.
