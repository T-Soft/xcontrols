Структура описательного XML отображения данных

 

<?xml version="1.0" encoding="UTF-8"?>
<view>
 <database>
         <table_name>skk_results</table_name>
 </database>
   
 <table>
         <style>
                 описание стилей и классов
         </style>

         <additional>
                 <field>

                         <source_name>имя дополнительного поля 1</source_name>                        

                 </field>

                 ...

                 <field>

                         <source_name>имя дополнительного поля n</source_name>                                

                 </field>
         </additional>
 </table>
   
 <data_row>
         <row>
                 <cell>

                         <block type="layout">
                        блок описания элемента разметки (hr / br)
                         </block>
                         <block type="data">         

                        блок описания поля базы данных
                         </block>

                         <block type="nodata">         

                        блок описания макросов и служебных полей
                         </block>
                 </cell>
         </row>         
 </data_row>
</view>

 

 <database>

- описание базы данных. Указывается единожды.

 <table>

- описание результирующей таблицы. Указывается единожды.

 <style>

- описание стилей и классов родительского элемента. Указывается единожды в каждом элементе <table>,<data_row>, <row>, <cell>, <block>. Необязательный элемент.

 <additional>

- описание дополнительных полей, включаемых в выборку.

 <data_row>

- описание ряда данных - строки из выборки по БД. Указывается единожды.

 <row>

- описание ряда результирующей таблицы. Ряд может содержать несколько ячеек. Указывается столько раз, сколько необходимо рядов таблицы.

 <cell>

- описание ячейки результирующей таблицы. Ячейка может содержать несколько блоков. Указывается столько раз, сколько необходимо ячеек таблицы.

 <block>

- описание содержимого ячейки результирующей таблицы. Блок является атомарной единицей и представляет собой одно поле базы данных. Указывается столько раз, сколько необходимо блоков в ячейке таблицы.

Внутри тэгов использование пробельных символов не допускается.
