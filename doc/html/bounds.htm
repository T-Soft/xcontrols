Ограничение вводимого значения

 

В описательном XML для некоторых полей возможно указание границ вводимых значений.

Границы служат для сообщения пользователю в реальном времени о логической корректности (или некорректности) вводимого значения.

При попадании в заданный интервал для данного поля будет выведено сообщение.

Границы задаются тегом <bounds min="" max="" type="" message=""/>

 

min

- значение левой границы диапазона значений

max

- значение правой границы диапазона значений

type

- тип ошибки при попадании в диапазон

message

- сообщение, выводимое при попадании в диапазон

По умолчанию границы диапазона включаются в сам диапазон. Чтобы исключить границу из диапазона, после значения ставится символ нижнего подчеркивания "_"

Например.

      max="12"        - включаем границу

 max="12_"         - не включаем границу

Атрибут type определяет тип границы и может принимать одно из следующих значений.

•info - служит для сообщения пользователю некритичной информации о вводимом значении. Не вызывает остановку отправки формы.

•warning - служит для сообщения пользователю о возможной ошибке ввода. Не вызывает остановку отправки формы.

•error - служит для сообщения пользователю об ошибке ввода. Вызывает остановку отправки формы.

В зависимости от типа границы меняется выделение контролируемого поля.

Тегов <bounds> может быть столько, сколько диапазонов требуется описать. Каждый тэг соответствует своему диапазону.

Пример.

    <bounds min="0_" type="error" message="Количество партий должно быть больше 0"/>
    <bounds min="0" max="99" type="info" message="Введено корректное количество партий"/>
    <bounds max="100_" type="warning" message="Количество партий достаточно велико, Вы уверены в корректности данного значения"/>
