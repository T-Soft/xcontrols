<style> Описание стилей родительского элемента

 

<style>...</style> - В данном элементе содержится описание стилей и классов, применяемых к родительскому элементу. 

Влияет только на отображение данных. Сутевая часть данных этим элементом не меняется.

 

В качестве родительского может быть использован один из следующих элементов: <table>, <data_row>, <row>, <cell>, <block>.

 

 <style>
         <decor>формализованное описание стилей начертания</decor>
         <inline>встроенные стили. нотация as-is</inline>
         <class>классы, налагаемые на элемент</class>
 </style>

Составляющие:

 

 <decor>

- элемент содержит формализованное описание стилей начертания данных элемента.

 <inline>

- элемент хранящий inline- стили. Содержимое переносится в код таблицы (элемент style="") как есть.

 <class>

- элемент хранящий классы, назначаемые родительскому элементу. Содержимое переносится в код элемента (элемент class="") как есть.

Для всех элементов таблицы, кроме элемента <block> содержимое элемента <style> переносится в код элемента верхнего уровня <table>, <tr> или <td>.

Для содержимого элемента <block> создается оборачивающий тэг <span> в код которого переносятся описанные в <style> стили и классы.

Если не указан элемент <style> для таблицы, по умолчанию к ней применяется стиль xc_table_view_default из style.css в директории системы. Для всех остальных элементов значений по умолчанию нет.

В общем виде:

 <tr class="классы из элемента class" style="[стили полученные путем преобразования формализованных описаний из элемента decor] [стили из элемента inline]">

Стили из элемента <inline> в случае совпадения перезапишут стили из элемента <decor>

Элемент <decor>

 Содержит описание начертания (жирное, курсив, подчеркнутое и т.д.) данных родительского элемента. Принимает следующие значения. Через / написаны возможные варианты нотации.
