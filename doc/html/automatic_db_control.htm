Контроль БД и данных

 

xС осуществляет автоматический контроль таблиы базы данных, с которой связан описательный XML.

Система контроля запускается при обнаружении любых обновлений файла описательного XML

Стадии контроля.

•автоматическое  создание таблиц

•контроль обновления количества / имен полей

•отчет системы контроля

•контроль на стороне пользователя

•контроль на стороне сервера

Автоматическое создание таблиц.

В случае отсутствия таблицы, в которую производится запись введенных пользователем значений происходит ее автоматическое создание на основе имен полей описательного XML.

Также создается таблица $имя_таблицы$$архивный_постфикс$ в которую вносятся предыдущие значения измененных записей основной таблицы БД.

Контроль обновления количества / имен полей.

В случае несовпадения количества и/или имени описанных в XML полей с полями базы данных, в которую производится запись введенных пользователем значений, БД будет дополнена полями, сгенерированными на основе имени отсутствующего поля XML. Также будут созданы поля базы для переданных методам скрытых полей (см. Статические методы )

При создании, все поля имеют тип VARCHAR (255).

Отчет системы контроля.

Вспомогательным элементом контроля является автоматически создаваемый файл *.flag , генерируемый рядом с описательным XML имеющим аналогичное имя. В данный файл заносятся результаты контроля структуры БД.

Удаление данного файла приведет к запуску системы контроля и созданию нового пустого файла *.flag содержащего результат последнего контроля.

Также xС осуществляет автоматический контроль введенных данных на стороне пользователя.

На основе типа элемента управления xC осуществляет автоматический контроль введенных данных (средствами javascript на стороне клиента) и в случае некорректности ввода сообщает об этом пользователю. (см. Контроль : клиент)

Финальная стадия контроля - контроль на стороне сервера.

Контроль на стороне сервера - необязательная операция, реализуемая каждым модулем системы для своего типа данных. (см. Контроль : сервер)
