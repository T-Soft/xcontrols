Препроцессор

 

Прежде чем начать интерпретацию XML документа, система xControls проводит его предварительные преобразования и компиляцию посредством препроцессора.

 

В задачи препроцессора входит удаление комментариев, включение содержимого дополнительных XML документов, исполнение макросов,

вызов нативных функций из XML а также замена специальных управляющих символов на их предустановленные значения, валидные для

последующего преобразования в HTML

 

Последовательность действий препроцессора такова:

 

1.Удаление комментариев

2.Включение содержимого из дополнительных XML (обработка элемента <include_xml>)

3.Исполнение макросов и вызов функций языка Parser 3

4.Замена специальных управляющих символов на их значения, валидные для последующего преобразования в HTML. [2.6]

5.Замена всех табуляций пробелами
