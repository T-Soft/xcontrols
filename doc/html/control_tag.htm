<control> Описание элемента управления

 

<control>...</control> - В данный контейнер заключается описание каждого элемента управления на форме.

Элементы управления делятся на значащие и незначащие (valueless). Значащие элементы управления передают данные из поля value на сервер. Valueless элементы никакого значения не передают и служат лишь для удобства работы со скриптами и формой, а также для визуального оформления.

Тэг используется столько раз, сколько элементов присутствует на форме, вне зависимости от их значимости.

Составляющие:
 

Тэг

Обязательность для элементов

Описание

 <control_group>

все

Класс элемента управления

 <control_type>

input

Тип элемента управления, описанного в классе

 <control_name>

все, кроме valueless

Имя элемента для передачи на сервер

 <control_id>

нет

Идентификатор элемента для привязки к скриптам. Если не указан - <control_name>_uid64

 <control_label>

все, кроме valueless

Текст рядом с элементом управления для вывода на экран.

 <control_sublabel 

  media="where_to_show"

 >

нет

Текст под текстом из <control_label> печатается <small>.Доступно для всех элементов, кроме составных.

Атрибут media отвечает за устройство на котором отображается данный текст.

 <control_default_value>

нет

Значение по умолчанию для элемента. Подставляется при генерации формы. Возможно использование макросов.

 <control_options>

select

Варианты выбора при генерации опций из известного набора данных. Составной элемент.

 <control_sql>

элементы, работающие с БД

Запрос на выборку значений для элементов, работающих с БД.

 <control_classes>

нет

Пользовательские классы. Добавляются в конец строки классов class=""

 <control_properties>

нет

Пользовательские свойства элемента.

 <bounds>

нет

Границы диапазона вводимых значений.

 <file_type>

file

Типы загружаемых файлов - пользовательские группы (не влияет на интерфейс).

 <file_ext>

file

Расширения загружаемых файлов (не влияет на интерфейс).

 <required/>

все, кроме valueless

Указывается, если поле обязательно к заполнению.

 <accept_initfile/>

все, кроме valueless и file

Указывается, если значение в поле можно загрузить из инициализирующего файла. (см. Инициализирующий файл)

 <readonly/>

все, кроме valueless

Указывается, если поле только для чтения. При наличии значения по умолчанию отображается как текст, а не как поле ввода. Доступно для всех элементов, кроме составных.

 

Элемент <control_label>

 

 Для включения в пояснительные надписи любого HTML-кода необходимо обрамить этот контент следующей конструкцией:

 

    <![CDATA[текст с HTML форматированием или контентом]]>

 

 Например:

 

                     <control_label><![CDATA[Файл архива со <b>сканированными</b> страницами экспертного заключения (размер до 30Мб)]]></control_label>

Элемент <control_group>

 Важно! Имя группы и имя модуля расширения класса-обработчика должны совпадать.

Элемент <control_default_value>

 Позволяет использовать в качестве значения по-умолчанию результаты выполнения функций-макросов или вызовов функций языка Prser3.

Элемент<control_options>

 Служит для хранения вариантов выбора для элементов типа select.

Элемент<control_sublabel media="where_to_show">

 Атрибут media="" может принимать следующие значения.

print

- текст отображается только при печати и не отображается на экране.

screen

- текст отображается только на экране и не отображается при печати.

атрибут не указан

- текст отображается на экране и, если не отработал сценарий noprint.js , при печати.

Например:
 

<control_options>
        <option>
                <label>option 1</label>
                <value>value_1</value>
        </option>
        <option>
                <label>option 2</label>
                <value>value_2</value>
        </option>
        <option>
                <label>option 3</label>
                <value>value_3</value>
        </option>
</control_options>

 Где <label> - информация, выдаваемая пользователю на страницу, а <value> - соответствующее ей значение, передаваемое на сервер в результате выбора.

Элемент <control_sql>

 Служит для хранения запроса на выборку данных для элементов типа select.

Например:
 

 <control_sql>SELECT `labels`,`values` FROM `test`</control_sql>

 Причем первый столбец выборки будет служить для заполнения информации, выдаваемой пользователю на страницу, а второй столбец выборки - для заполнения данных, отправляемых на сервер как результат выбора конкретной опции.
