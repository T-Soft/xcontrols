Контроль : сервер

 

На стороне сервера, перед сохранением или изменением данных в базе, для каждого значащего элемента управления вызывается метод

 

 ^xcontrols:db_preprocess_$sControl_group[value].

 

В данном методе может быть реализован форматно-логический контроль переданного значения и возвращена служебная информация о внесении изменений в базу или ошибках.

 

Методы, вызывающие контроль:

 

 ^xcontrols:store_data[]

 

 ^xcontrols:update_entry[]

 

В случае успешного внесения в базу или изменения записи, возвращают хеш:

 

$h[
      $.cnt_err(0)
      $.hErr[]
      $.id(12)
      $.uuid[DF2C94CD-2712-4160-8BF0-EA06F25A2E92]
]

 

Где:

                                                         

$.cnt_err(0)

- количество ошибок. В случае успешного внесения в базу - 0

$.hErr[]

- хеш с развернутой информацией об ошибках. В случае успешного внесения в базу пуст.

$.id(12)

- идентификатор только что внесенной записи.

$.uuid[]

- uuid только что внесенной записи.

 

Важно (!): ключи хеша имеют именно такие имена.

 

В случае возникновения ошибок внесения в базу или изменения записи, возвращают хеш:

 
$h[
   $.cnt_err(2)
   $.hErr[
      $.1[
         $.err_fieldname[file_test]
         $.message[FILE VALUE TYPE OR EXTENSION MISMATCH]
         $.err_module[FILE.TYPE_MISMATCH]
         $.err_main[MAIN.INVALID_VALUE]
      ]
      $.2[
         $.err_fieldname[file_test_2]
         $.message[FILE VALUE TYPE OR EXTENSION MISMATCH]
         $.err_module[FILE.TYPE_MISMATCH]
         $.err_main[MAIN.INVALID_VALUE]
      ]
   ]
   $.failed_id[B795CD36-126A-4E29-AE11-407C79C6D91A]
]
 
Где:                                                        

 

$.cnt_err(0)

- количество ошибок.

$.hErr[]

- хеш с развернутой информацией об ошибках. Ключи хеша - порядковые номера ошибок, начиная с 1. Значения - хеши.

$.err_fieldname

- имя поля, в котором обнаружена ошибка.

$.message

- развернутое сообщение об ошибке.

$.err_main

- сообщение об ошибке, исходящее от ядра системы. (см. Ошибки ядра)

$.err_module

- сообщение об ошибке, исходящее от соответствующего модуля. (см. Ошибки модулей). Более подробное, чем err_main.

$.failed_id

- идентификатор временного файла сериализации формы.

 

Важно (!): ключи хеша имеют именно такие имена.
