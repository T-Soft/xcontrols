file. Загрузка файла

 

Поле для присоединения и отправки файла.

 

Корректность введенного значения (совпадение с типом или расширением) проверяется на сервере. В случае ошибки форма возвращается пользователю для исправления.

 

         <control_group>file</control_group>

Пример.
 

<control>
        <control_group>file</control_group>
        <control_name>tst_file</control_name>
        <control_id>file_id</control_id>
        <file_ext>file,extensions,comma,separated</file_ext>
        <control_classes>foo</control_classes>
</control>

Элементы <control_id> и <control_classes> являются необязательными.
 
Контроль файла на соответствие указанному типу осуществляется на стороне сервера. В случае несовпадения расширения с указанными в тэге <file_ext> форма возвращается пользователю для исправления.

Поддерживаемые расширения а также максимально допустимый размер файла (соответствующий максимальному размеру POST-запроса) выводятся под соответствующей полю с файлом поясняющей надписью.

Сохранение загруженного файла происходит после прохождения форматно-логического контроля. Директория сохранения определяется полем инициализации $.binary_path[/folder_to_save_files/]

Указанный каталог создается автоматически и имеет следующую структуру.

$.binary_path/  
|__ <control_name>/ 
  |___index.cfg 
  |___год/ 
    |______месяц/ 

              |_______день/

                |_______<UUID>.filename

               |_______<UUID>.file

Поля ввода типа file имеют встроенный контроль максимального размера. В случае, если суммарный размер всех загруженных файлов превышаем максимальный размер POST, система выдает ошибку и соответствующее сообщение.
