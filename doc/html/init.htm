init. Инициализация модуля создания форм

 

При инициализации модуля необходимо передать следующие опции.

 

 ^xcontrols:form_init[
         $.user_xml_folder_path[/project_root/.xcontrols_forms/]
         $.form_target[/redirect_page/]
         $.project_nick[project]
         $.is_save_xml(false)
         $.binary_path[/folder_to_save_files/ or hash]
         $.temp_path[/folder_to_save_temp_files/]
         $.uri_show_form[/page_to_handle_form_show/]
         $.uri_save_data[/page_to_handle_data_storage/]

              $.enable_initfile(true)

              $.is_static_form(true)

         $.is_debug_form_xml(true)

         $.BP[/base_web_path/]

         $.table_charset[

                 $.charset[]

                 $.collate[]

         ]
 ]

 

Где:

 

$.user_xml_folder_path

- путь к директории с описательным XML.

Важно: каждый описательный XML также должен находиться в своем каталоге.

$.form_target

- путь перенаправления после завершения работы с формой.

$.project_nick

- имя проекта (должно совпадать с директорией расположения описательного файла form.xml ).

$.is_save_xml

- если выставлено true - будет сохранен XML документ после отработки препроцессора т.е. после всех обработок и исполнения макросов.

$.binary_path

- путь сохранения файлов переданных через элемент <file>. Возможно задание в виде строки или хеша.

$.temp_path

- путь сохранения временных файлов модуля. По умолчанию /temp/.

$.uri_show_form

- адрес документа, отвечающего за показ формы. Если не указан - перенаправления на этот документ при ошибке не произойдет,

$.uri_save_data

- адрес документа, отвечающего за сохранение или обновление данных в базе. По умолчанию store.html.

$.enable_initfile

- при передаче true будет выведена дополнительная форма для загрузки инициализирующего файла. (см. Инициализирующий файл). По умолчанию false.

$.is_static_form

- при передаче true будет скомпилирован файл form_static.htm, содержащий статический html формы. Данный файл будет сохранен рядом с описательным XML проекта. При последующем вызове модуля с данным параметром, будет выведен он. По умолчанию false.

В случае, когда файл form_static.htm отсутствует, будет произведена обычная, динамическая генерация формы, а файл будет пересобран и вновь сохранен. [2.4]

$.is_debug_form_xml

- при передаче true включает вывод отладочной информации по xml - на данный момент показывает имена элементов управления в форме. По умолчанию false.

$.BP

- веб-путь до проекта. По умолчанию /

$.table_charset

- данные по кодировкам и сравнениям для создания основной и архивной таблиц.

    $.charset

- данные по кодировке. По умолчанию utf8

    $.collate

- данные по сравнению. По умолчанию utf8_unicode_ci

 

Важно: Все пути передаются с закрывающим " / "

Важно: Все ключи хеша имеют именно такие имена.

 

Тип возвращаемого значения:        void.

 

Параметр $binary_path[] :

 

 Начиная с версии [2.6] система поддерживает назначение имен путям до директорий для сохранения файлов. Эти директории называются хранилищами данных (data_storage).

 Для того, чтобы создать именованное хранилище необходимо в параметре $binary_path[] передать хеш вида: $.<имя_хранилища>[<путь>].

 Если такой функционал не требуется а также в целях обратной совместимости с предыдущими версиями, допускается указание строки в параметре $binary_path[].

 В таком случае будет создано единственное хранилище с именем default.

 

Параметр $table_charset[] :

 

 Начиная с версии [2.6] система поддерживает указание кодировки и сравнений для создания основной и архивной таблиц. По умолчанию это - utf8.

 

 Пример указания кодировок cp1251:

 

$.table_charset[
   $.charset[cp1251]
   $.collate[cp1251_general_ci]
]
