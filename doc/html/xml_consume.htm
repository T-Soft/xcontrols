<include_xml> Включение XML из внешнего файла

 

Начиная с версии 2.1 поддерживается возможность включать в главный описательный файл form.xml содержимое других (дополнительных) xml документов при условии совпадения структуры элементов.

 

Структура элемента описания поглощаемого файла:

 

   <include_xml>
                 <prefix>префикс имен элементов дополнительного XML</prefix>

                 <file>путь к дополнительному XML</file>

                         <readonly/>
   </include_xml>

Тэг используется столько раз, сколько файлов необходимо поглотить.

Составляющие:
 

Элемент

Описание

 <prefix>

Префикс, который будет прибавлен к содержимому элемента <control_name> каждого описанного в дополнительном xml элемента управления.

Например: <control_name>foo</control_name> + <prefix>bar</prefix> = <control_name>foo_bar</control_name>

 <file>

Путь к дополнительному xml относительно главного form.xml. Например: foo.xml или ../bar.xml

 <readonly/>

Все добавленные элементы управления будут помечены тэгом <readonly/> как "только для чтения".

 

Важно (!):

 

 Поглощаемый XML должен включать только описание элементов управления <control>. Контроль правильности не осуществляется, заголовки не игнорируются.

 Весь текст переносится как есть с заменой имен на иена с префиксом.

 

Содержимое поглощаемого файла будет перенесено в то место основного XML , где был указан элемент <include_xml>.
